# ai-home_assistant_linux
# 🧠 Проект «Джарвис» (Local Voice AI)

Полностью локальный, приватный и автономный голосовой ИИ-ассистент для умного дома на базе Home Assistant. Никаких облаков, платных подписок и сбора ваших данных.

## 🎯 О проекте
«Джарвис» — это распределенная система голосового управления. Она состоит из мощного «Мозга» (сервера), который берет на себя работу с нейросетями, и легких «Ушей» (клиентов/колонок), которые можно разместить на слабом железе (Raspberry Pi) по всему дому. 

В качестве интеллекта используется локальная LLM, которая способна как управлять устройствами Home Assistant, так и поддерживать свободный диалог.

## 🔄 Архитектура и Data Flow

Система работает через **Wyoming Protocol** и состоит из двух узлов, соединенных локальной сетью или защищенным VPN-туннелем (Tailscale).

1. **Фоновое слушание:** Клиент (OHF Linux Voice Assistant) непрерывно анализирует звук через `openWakeWord`.
2. **Активация:** При слове «Джарвис» открывается TCP-стрим на сервер.
3. **STT (Распознавание):** `Faster-Whisper` на сервере переводит речь в текст.
4. **Логика:** - Простые команды («Включи свет») обрабатываются мгновенно ядром HA.
   - Сложные («Что надеть в дождь?») уходят в `Ollama` (любая модель).
5. **TTS (Синтез речи):** `Piper` генерирует аудио-ответ.
6. **Воспроизведение:** Ответ стримится обратно на клиент и звучит из динамика.

## 🛠 Предварительные требования

**Для Сервера («Мозга»):**
* ОС: Linux (Ubuntu Server / Debian)
* RAM: 4+ ГБ (рекомендуется 6–8 ГБ для Ollama)
* Docker и Docker Compose
* *(Опционально)* Tailscale для удаленного доступа

**Для Клиента («Колонки»):**
* Устройство: Raspberry Pi 3/4/5 или любая Linux-машина (VM)
* Микрофон и динамик (USB или 3.5mm)
* Docker и Docker Compose

## 🚀 Развертывание системы

### 1. Запуск Сервера
1. Перейдите в папку сервера:
   ```bash
   cd server
   
2. Поднимите стек контейнеров:
   ```Bash
   docker compose up -d

3. Скачайте LLM-модель для Ollama:
   ```Bash
   docker exec -it ollama ollama run <название модели>
   
4. Настройте интеграции Wyoming и Ollama в веб-интерфейсе Home Assistant (http://<IP_СЕРВЕРА>:8123).

### 2. Запуск Клиента
1. Перейдите в папку клиента на устройстве-колонке:

   ```Bash
   cd client
   
2. Проверьте индекс вашего микрофона командой arecord -l и при необходимости обновите параметр --mic-device plughw:X,Y в docker-compose.yml.

3. Поднимите клиент:

   ```Bash
   docker compose up -d
   
4. В Home Assistant добавьте интеграцию Wyoming Protocol, указав IP-адрес клиента и порт 10420.

## 🌐 Использование вне дома (VPN)
Для работы клиента вне локальной сети используется Tailscale.

Установите Tailscale на сервер и клиент.

В файле client/docker-compose.yml укажите Tailscale IP сервера (начинается на 100.x.x.x) вместо локального адреса.

## 📚 Стек технологий
- Умный дом: Home Assistant

- Ассистент: OHF Linux Voice Assistant

- Wake-word: OpenWakeWord

- STT (Распознавание): Wyoming Faster-Whisper

- LLM (Интеллект): Ollama + любая модель

- TTS (Синтез речи): Wyoming Piper
